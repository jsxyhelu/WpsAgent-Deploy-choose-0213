# WpsAgent 项目开发经验总结

## 项目概述
WpsAgent 是一个 WPS Office 加载项，集成了 AI 功能来辅助用户处理文档。支持云端模型（DeepSeek、OpenAI 等）和本地大模型（Ollama、vLLM 等）。

### 4. 帮助关闭功能修正
**问题**: 初始实现使用 `window.close()` 导致退出整个 WPS 应用。

**修正方案**:
- 创建 `closeHelp()` 函数
- 从 PluginStorage 获取帮助面板 ID
- 获取对应的 TaskPane 对象
- 将 `Visible` 设置为 `false`，只关闭帮助面板
- 添加错误处理和日志记录

**经验教训**:
- ✅ 理解 WPS API 的 TaskPane 机制
- ✅ 使用 PluginStorage 管理 Panel ID
- ✅ 区分关闭面板和退出应用的操作
- ✅ 添加错误处理，避免影响用户体验

### 5. JavaScript 模块加载问题修复
**问题**: 浏览器环境不支持 Node.js 的 `require()` 语法，导致按钮功能失效。

**根本原因**:
- JavaScript 文件使用了 `require()` 导入模块
- 浏览器环境不支持 CommonJS 模块系统
- 导致所有依赖模块的功能无法正常工作

**修复方案**:
- 移除所有 JavaScript 文件中的 `require()` 语法
- 改为全局函数模式，直接在浏览器中定义
- 修复的文件：
  - `logger.js` - 日志系统
  - `config.js` - 配置管理
  - `api.js` - API 调用
  - `document.js` - 文档操作
  - `tabManager.js` - Tab 管理
  - `taskpane_new.js` - 任务面板
- 修复的语法错误：
  - Promise 括号问题
  - HTML 字符串闭合问题
  - if 语句语法错误

**经验教训**:
- ✅ 浏览器环境不支持 Node.js 的 `require()`
- ✅ 需要使用 ES6 模块或全局函数
- ✅ 语法错误会导致整个文件无法加载
- ✅ 应该在开发环境中测试浏览器兼容性

## 关键经验教训

### 1. WPS API 使用
- **TaskPane 管理**: 使用 `PluginStorage` 存储 Panel ID，通过 ID 获取和操作面板
- **文档操作**: 使用 `Application.ActiveDocument` 访问当前文档
- **配置存储**: 使用 `PluginStorage` 持久化用户配置

### 2. 浏览器环境限制
- **模块加载**: 在浏览器环境中，`require()` 的加载顺序很重要
- **全局变量**: 避免过度使用全局变量，使用模块导出
- **错误处理**: 所有异步操作都应该有 try-catch 包裹

### 3. 用户体验设计
- **操作反馈**: 每个操作都应该有明确的反馈（成功/失败）
- **步骤清晰**: 使用编号列表和图标标识操作步骤
- **示例丰富**: 提供多种示例（对话、文本、场景）
- **渐进学习**: 从初级到高级，循序渐进

### 4. 安全性考虑
- **API Key 保护**: 使用环境变量，不硬编码在代码中
- **本地模型优势**: 无需网络传输，保护用户隐私
- **配置验证**: 所有用户输入都应该进行验证

### 5. 代码组织
- **单一职责**: 每个模块只负责一个功能领域
- **依赖管理**: 明确模块间的依赖关系
- **接口设计**: 提供清晰的公共接口
- **日志系统**: 统一的日志记录，便于调试和问题排查

## 常见问题排查

### 问题 1: 插件无法加载
**可能原因**:
- WPS Office 版本不支持 JSAPI
- manifest.xml 路径不正确
- 服务器未运行

**解决方案**:
1. 检查 WPS Office 版本
2. 确认 manifest.xml 路径正确
3. 确认服务器运行在 http://localhost:3889

### 问题 2: API 调用失败
**云端模型**:
- 检查 API Key 是否正确
- 检查网络连接是否正常
- 检查 API URL 是否有效

**本地模型**:
- 检查本地服务是否启动
- 检查 API 地址是否正确
- 检查模型是否已下载
- 检查端口是否被占用

### 问题 3: 帮助面板无法关闭
**可能原因**:
- PluginStorage 中没有保存帮助面板 ID
- TaskPane 对象获取失败

**解决方案**:
1. 检查浏览器控制台错误信息
2. 清除 PluginStorage 中的旧数据
3. 重新加载帮助面板
